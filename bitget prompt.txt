PROJECT: Adaptive Consensus Trading Agent (Bitget AI Hackathon S1 — Trading Agent track)
 
GOAL
Build a full-stack AI trading agent that:
1. Pulls live market perception from Bitget's 5 Skill Hub skills (macro-analyst, 
   sentiment-analyst, technical-analysis, market-intel, news-briefing) for BTC, ETH, 
   and one meme coin
2. Classifies each asset's market regime (trending / ranging / unclear) from 
   technical-analysis output
3. Runs a weighted consensus vote across all 5 skills (bullish/bearish/neutral score 
   each), with weights that adjust over time based on which skills' calls proved 
   correct historically
4. Picks a strategy mode per regime (trend-follow when trending, mean-revert when 
   ranging, stay flat when unclear) and combines it with consensus conviction to 
   decide direction
5. Generates a natural-language rationale for every decision (use MuleRun API — 
   sign up at mulerun.com, claim 2000 credits at credits.mule.page with code 
   0526BITGET — call their chat completion endpoint, NOT OpenAI)
6. Logs every decision (timestamp, asset, regime, skill scores, weights, direction, 
   conviction, rationale, simulated entry price, simulated PnL on exit) as a 
   paper-trading record — write to a JSON file committed to the repo so it's a 
   public, verifiable usage/trading log
7. Renders a live dashboard showing: current regime per asset, skill weight 
   evolution over time (chart), decision feed with rationale, and cumulative 
   simulated PnL
 
SETUP STEPS
1. Install getagent: npm install -g @bitget-ai/getagent-skill
2. Install Bitget MCP: npx bitget-hub install --target codex
3. Configure Bitget API keys as env vars (BITGET_API_KEY, BITGET_SECRET_KEY, 
   BITGET_PASSPHRASE) — read/sim trade permissions only, no live trade permission
4. Configure MuleRun API key as env var (MULERUN_API_KEY)
5. Scaffold a Next.js 15 + TypeScript app (App Router, Tailwind)

DECISION ENGINE LOGIC
- Each skill returns a score from -1 (bearish) to +1 (bullish) per asset
- Maintain a weights object per skill, starting equal (0.2 each), persisted in 
  data/weights.json
- After each simulated trade closes, compare each skill's directional call against 
  actual outcome; nudge that skill's weight up if correct, down if wrong (simple 
  exponential moving average adjustment, clamp between 0.05 and 0.5, renormalize 
  to sum to 1)
- Regime detection: use technical-analysis indicators (e.g. ADX-style trend 
  strength + volatility bands) to classify trending/ranging/unclear
- Final signal = weighted sum of skill scores; threshold for action vs flat should 
  be conservative — only act when |weighted score| crosses a meaningful threshold 
  AND regime supports that strategy type
- Every decision and its full reasoning must be logged even if the action is "flat"
